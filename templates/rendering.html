What is search engine rendering?

Once a search engine bot has crawled a page they will render it in order to run the code contained within the html in order to assess and understand the content.

When rendering a page, search engine crawlers will collect information on the quality of the site and use it in order to determine the rank of the page in search engine results.

All HTML pages have an initial state, which is the when is the first response. This is followed by the rendered HTML, also known as the DOM (Document Object Model). This represents the initial state once any JavaScript has been added to it. You can view a page's DOM in your browser developer tools.

Search engines such as Google are only able to index what they can render from a page, so if it can't be rendered then it can't be used to understand the content. Typically this issue occurs if JavaScript is used to add important information to a page. Therefore you need to be aware of how content is being served on your site and ensure nothing is blocked from rendering.

Remember if content is imperative for users this should take precedent over any fancy, JavaScript led, features.
